# Frontend Skill

当用户需要真正开始做前端代码实现时，使用这个技能。

## 目标

把页面设计、产品结构和交互方案，落成清晰、可扩展、可维护的前端实现。

## 适用场景

- 根据现有产品方案开始搭前端
- 为新项目建立前端目录结构
- 实现页面、组件、布局、状态流转
- 为查询、结果、详情、调试、设置等页面做信息组织
- 为后续扩展预留组件和页面结构

## 默认工作流

1. 明确前端使用的技术栈和边界
2. 拆分页面级结构
3. 设计布局与导航骨架
4. 抽取基础组件与复用模块
5. 组织数据层、状态层和交互层
6. 完成首版实现并验证可运行性

## 实现原则

- 优先搭好页面骨架与导航关系
- 优先保证主流程可用，再补细节
- 组件抽象只做必要层级，不过早泛化
- 保持文件职责单一
- 视觉实现应与项目定位一致，不使用通用模板感过强的页面

## 对“从 0 到 1 新建前端”的要求

如果当前仓库还没有前端，默认需要覆盖：

- 前端目录结构
- 应用入口
- 路由结构
- 页面骨架
- 基础布局
- 设计变量或主题变量
- 关键页面占位与主流程

## 面向这个项目的优先页面类型

如果项目属于“索引 / 检索 / 证据 / 问答 / 调试”类型，优先考虑：

- 首页
- 构建索引页
- 检索与提问页
- 证据结果页
- 调试与 trace 页
- 评测或回归结果页

## 与设计衔接

如果需求还停留在“先想清楚前端长什么样”，应先使用 `frontend_design`，再进入这个技能。
