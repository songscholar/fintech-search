# Frontend Design

当用户要为一个还没有前端的项目设计界面时，使用这个技能。

## 目标

围绕“先定义前端要解决什么问题，再决定页面长什么样”来推进设计，避免直接陷入零散页面或随意堆组件。

## 适用场景

- 项目还没有前端，需要从 0 到 1 设计
- 需要定义首页、工作台、结果页、详情页、调试页等页面结构
- 需要先做产品信息架构、用户路径、模块拆分
- 需要先定视觉方向，再进入实现

## 工作方式

默认按下面顺序推进：

1. 先理解项目能力边界
2. 明确核心用户和高频任务
3. 设计信息架构与导航结构
4. 设计关键页面与页面关系
5. 定义视觉方向、设计原则、组件风格
6. 输出可执行的前端设计方案

## 输出要求

输出应尽量包含这些内容：

- 前端目标用户
- 核心使用场景
- 页面地图
- 主导航和次导航
- 关键页面说明
- 每个页面的核心模块
- 页面之间的数据与操作流转
- 视觉方向建议
- 适合的技术落地建议

## 设计原则

- 先产品结构，后视觉表现
- 先高频任务，后边角功能
- 先页面角色，后组件细节
- 优先让界面帮助理解系统能力，而不是只展示数据
- 设计应服务于真实使用，不做空洞的展示型页面

## 针对无前端项目的特别要求

如果项目目前没有前端，默认优先产出：

- 一个首页或 Landing Page 的定位
- 一个主工作台或主搜索页
- 一个结果页或详情页
- 一个调试/诊断页（如果项目有检索、索引、问答、日志、trace 等能力）
- 一个设计语言概要

## 与实现衔接

如果用户后续要开始编码，应继续把方案交给 `frontend_skill` 进行页面结构和工程实现。
