### 报错返回

- **参数**: `[错误号]{[参数列表]}`
- **适用类型**: 服务,函数,原子服务,原子函数,因子服务,因子函数

```
[报错返回][LDP_MDB_MOD_FAIL][@table_name, @fund_account]
<W>[报错返回][LDP_MDB_MOD_FAIL][@table_name, @fund_account]
```

**说明**: 【参数说明】
·错误号：元数据/标准错误中定义的错误ID
·参数列表：@variable,...
【备注】

**标记说明**: W:报错信息写分布式日志。报错返回宏，在返回报错信息的同时，核心会记录分布式日志(dlog日志)。但是在上层调用中使用了继续执行宏之后，会导致报错信息无法记录日志。若这种场景下需要记录分布式日志，则建议使用W标记。

### 手工打包体清空

- **参数**: `{[打包变量]}`
- **适用类型**: 服务,函数,原子服务,原子函数,因子服务,因子函数

```
[手工打包头][fund_account, money_type]
  if(1)
  {
        [手工打包体][@fund_account, @money_type]  
  }
  else
  { 
	[手工打包结束]
	[手工打包体清空]
	[手工打包体][@fund_account, @money_type]
  }
  [手工打包结束]
```

**说明**: 【参数说明】
 ·打包变量：HsPacker类型的变量
 1.[手工打包体清空]必须与[手工打包头]、[手工打包体]、[手工打包结束]在同一个函数中使用
 2.[手工打包体清空]只有在前面有[手工打包头]使用的情况下，用这个宏才不报错，并且和[手工打包头]变量需要匹配，这里的变量应该指"打包变量",即打包时用到的packer
 3.[手工打包体清空]的作用是打包恢复到[手动打包头]状态，避免如代码示例场景[手工打包体]宏误报

```

**说明**: 【参数说明】

【备注】
·必须与宏[事务处理结束]配套使用。
·从宏[事务处理开始]到[事务处理结束]之间被调用的函数，不允许包含宏[事务处理开始]，否则会导致当前事务被回滚。

·不建议在一个事务中对太多的记录进行增、删、改的操作。

### 获取记录

- **参数**: `[{上级.}对象{(索引)}][{条件列表}]{[输出列表]}或者是[子对象.父级对象][{条件列表}]{[输出列表]}`
- **适用类型**: 服务,函数,原子服务,原子函数,因子服务,因子函数

```
[获取记录][FundAccountCtrl.StockAccountCtrl][exchange_type=@exchange_type][main_flag=@main_flag, stock_account=@stock_account]
```

**说明**: 【参数说明】
·上级：未指定表示从全局容器获取（非全局容器，开发工具生成代码时会报错）
·对象：具体获取的记录类型
·条件列表：field_name=@variable,...
·输出列表：field_name=@variable,...，用于在获取字段同时，获取特定的字段内容。
【备注】
·支持传入的参数比索引字段多，超过的字段按条件进行比较，获取记录的D标记不支持该功能

**标记说明**: Z:生成函数代码，函数报错时不生成 返回语句，上下文错误号置为0。
M:生成函数代码，函数报错时不生成返回语句。
F:生成代码时，如果输出参数为*，则对除了条件列表中的表字段进行赋值。
D:只判断记录是否存在，不进行数据拷贝。

### 遍历记录开始

- **参数**: `[{上级.}对象(索引)][{条件列表}]{[输出列表]}`
- **适用类型**: 服务,函数,原子服务,原子函数,因子服务,因子函数

```
[遍历记录开始][branch(idx_now_entrust_no)]
[遍历记录结束]

//(非唯一索引) 匹配全部索引字段

[遍历记录开始][fundaccount(idx_fundaccount)][fund_account > @fund_account]

```

**说明**: 【参数说明】

·对象：具体遍历的记录类型
·索引：上级关联对象的索引或全局索引
·条件列表：field_name=@variable,...
·输出列表：field_name=@variable,...
【备注】
·必须和宏[遍历记录结束]配套使用
·根据条件是否包含索引的全部索引字段，决定全量扫描，或索引查找（必须包含第一个索引字段）
·支持在[遍历记录开始]前前缀<R>标志，以示逆序遍历
.唯一索引遍历条件支持>,>=,<,<=的使用

**标记说明**: R:逆序遍历。
E:对于非唯一索引或分级索引的情况下，索引条件和索引字段不匹配，开发工具不报错。

<Q>[数据回库][fundaccount]["USERDB"]["fundaccount"][][] //快速回库模式
	[数据回库][fundaccount]["USERDB"]["insert into fundaccount(fund_account) values(:fund_account) "][][] //普通回库模式
<B>[数据回库][branch]["usersvr"]["branch"][][] //绑定变量回库模式
<P>[数据回库][branch(idx_branch)]["usersvr"]["branch"][][]

<BU>[数据回库][branch(idx_branch)]["usersvr"]["merge into branch t1 using (select :branch_no branch_no, :now_entrust_no now_entrust_no from dual) t2 on(t1.branch_no = t2.branch_no) when matched then update set t1.now_entrust_no = t2.now_entrust_no when not matched then insert(branch_no,now_entrust_no) values(t2.branch_no,t2.now_entrust_no);"][][]
//O标记
<O>[数据回库][branch]["usersvr"]["update set now_entrust_no = :now_entrust_no where branch_no = :branch_no"][][]
<O>[数据回库][branch]["usersvr"]["insert into branch(branch_no, now_entrust_no) values(:branch_no, :now_entrust_no)"][][]
<O>[数据回库][branch]["usersvr"]["delete from branch where branch_no = :branch_no"][][]

//表名为变量
[数据回库][@table_name]["usersvr"][@sql_str][][]
```

**说明**: 【参数说明】
	上级：需要回库的内存表对象的父对象
	对象名：需要回库的内存表对象，支持传变量
	索引：指定回库的记录为该索引中的记录
	数据库连接名：需要落地的文件名或者回库的物理数据库逻辑名
	回库语句：将内存表的内容插入或者更新到物理数据库的SQL语句,支持传变量。
	过滤条件：将符合此条件的数据回库，过滤条件使用C代码的判断格式，如
 			 1）表字段过滤：branch_no = @branch.entrust_no 其中branch_no 与entrust_no均为表对象字段;
  			 2）字符串过滤：account_name=="abc";
  			 3）多个过滤条件使用 && 或|| 进行组合，例如branch_no > 10 && branch_no <15
	记录数：输出实际回到物理数据库的记录个数
【备注】
	1.A标记下物理数据库表的第一个字段必须为uft_rowid。
	2.回库语句使用Oracle的merge语句时必须打U标记。
	3.Oracle绑定变量回库性能更好，Mysql性能不佳不建议使用，Oracle10g和达梦数据库不支持绑定变量回库。
	4.U标记下的“回库语句”要求执行插入或者更新的物理数据库表中必须存在主键或者唯一索引。“回库语句”示例如下：
	  1）mysql的insert示例: 
		 insert into account(id, name, age) values(:id, :name, :age) on duplicate key update name = values(name), age = values(age)
	  2）oracle的insert示例：
		 select count(*) into rowcount from dual where exists(select 1 from account where id = :id);
		 if rowcount = 0 then
			insert into account(id, name, age) values(:id, :name, :age);
		 else
			update account set name = :name, age = :age where id = :id;
		 end if;
	  3）oracle的merge示例：
		 merge into account t1 using (select :id id, :name name, :age age from dual) t2 on(t1.id = t2.id) 
		 when matched then update set t1.name = t2.name, t1.age = t2.age 
		 when not matched then insert(id, name, age) values(t2.id, t2.name, t2.age);
	  4）Oracle的merge语句的性能比insert语句的好，即3）的性能比2）的性能好。 
      5. B标记和U标记不能与P标记或Q标记一起使用。
      6. 表名使用变量名的场景，不支持上级.对象名(索引)写法和加过滤条件的写法

**标记说明**: A:使用uft_rowid索引模式进行回库。普通回库模式下的回库语句中必须指定字段uft_rowid且为第一个字段。
B:绑定变量回库模式,达梦数据库不支持。
P:并发回库模式，该标记只在oracle的快速模式下有效，解决并发回库时等待锁表超时的场景。但使用P标记后，需要手工重建索引。
Q:快速回库模式，该模式会将内存表中与物理数据库表中相同的字段回到物理数据库表中。
U:在非快速回库模式下使用，支持对回库记录的插入更新(如果记录存在则更新，不存在则插入)功能。
O:表示回一条数据,这时sql语句中支持写update或者delete语句

	过滤条件：将符合此条件的数据回库，过滤条件使用C代码的判断格式，如
 			 1）表字段过滤：branch_no = @branch.entrust_no 其中branch_no 与entrust_no均为表对象字段;
  			 2）字符串过滤：account_name=="abc";
  			 3）多个过滤条件使用 && 或|| 进行组合，例如branch_no > 10 && branch_no <15
	记录数：输出实际回到物理数据库的记录个数
【备注】
	1.A标记下物理数据库表的第一个字段必须为uft_rowid。
	2.回库语句使用Oracle的merge语句时必须打U标记。
	3.Oracle绑定变量回库性能更好，Mysql性能不佳不建议使用，Oracle10g和达梦数据库不支持绑定变量回库。
	4.U标记下的“回库语句”要求执行插入或者更新的物理数据库表中必须存在主键或者唯一索引。“回库语句”示例如下：
	  1）mysql的insert示例: 
		 insert into account(id, name, age) values(:id, :name, :age) on duplicate key update name = values(name), age = values(age)
	  2）oracle的insert示例：
		 select count(*) into rowcount from dual where exists(select 1 from account where id = :id);
		 if rowcount = 0 then
			insert into account(id, name, age) values(:id, :name, :age);
		 else
			update account set name = :name, age = :age where id = :id;
		 end if;
	  3）oracle的merge示例：
		 merge into account t1 using (select :id id, :name name, :age age from dual) t2 on(t1.id = t2.id) 
		 when matched then update set t1.name = t2.name, t1.age = t2.age 
		 when not matched then insert(id, name, age) values(t2.id, t2.name, t2.age);
	  4）Oracle的merge语句的性能比insert语句的好，即3）的性能比2）的性能好。 
      5. B标记和U标记不能与P标记或Q标记一起使用。
      6. 表名使用变量名的场景，不支持上级.对象名(索引)写法和加过滤条件的写法

**标记说明**: A:使用uft_rowid索引模式进行回库。普通回库模式下的回库语句中必须指定字段uft_rowid且为第一个字段。
B:绑定变量回库模式,达梦数据库不支持。
P:并发回库模式，该标记只在oracle的快速模式下有效，解决并发回库时等待锁表超时的场景。但使用P标记后，需要手工重建索引。
Q:快速回库模式，该模式会将内存表中与物理数据库表中相同的字段回到物理数据库表中。
U:在非快速回库模式下使用，支持对回库记录的插入更新(如果记录存在则更新，不存在则插入)功能。
O:表示回一条数据,这时sql语句中支持写update或者delete语句

### 获取分片号

- **参数**: `[{分片规则名称}][分片键值][扩展键值][分片号]`
- **适用类型**: 服务,函数,原子服务,原子函数,因子服务,因子函数

```
[获取分片号][][ShardingKeys= @fund_account][SystemNo=@system_no][node_no = @node_no]
[获取分片号][@rule_name][ShardingKeys="12"][SystemNo=2][node_no = @node_no]
```

**说明**: 【参数说明】
.分片规则名称：可选，若不填，则默认Default。支持常量，整型和变量。
.分片键值：字符串类型，必填，用来匹配分片规则，格式为 key=value，其中value支持常量，整型和变量。key对应LocateRules标签下ShardingKeys配置的字符串，value传递ShardingKeys配置的字符串对应的值。例如根据账户查找分片号，则key传递account，value传递account的值
.扩展键值：格式为key=value，value目前只支持整型。key对应LocateRules->Rules标签下的SystemNo，value传递SystemNo的值。

.分片号：存储获取到的分片号变量, 格式固定为node_no = @variable
