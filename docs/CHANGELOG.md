## 2025-04-15

### 新功能

- **多跳调用链支持**
  - 从固定的 2 跳扩展为支持最多 10 跳调用链
  - 使用 BFS 算法进行多级遍历
  - 不同跳数有不同的权重（1-3跳强信号，4-6跳中等信号，7-10跳弱信号）
  - 追踪 outgoing（调用他人）和 incoming（被调用）两个方向的调用链

### 重排权重

| 跳数 | 权重 |
|-----|------|
| 1 | 3.0 |
| 2 | 1.5 |
| 3 | 1.0 |
| 4 | 0.6 |
| 5 | 0.4 |
| 6 | 0.25 |
| 7 | 0.15 |
| 8 | 0.1 |
| 9 | 0.05 |
| 10 | 0.02 |

### 代码改动

- `_procedure_call_neighbors()`: 支持 `max_depth` 参数，默认 10
- 重排逻辑：按跳数分配不同权重
- 证据组装：包含 `hop_depth` 和 `direction` 信息
