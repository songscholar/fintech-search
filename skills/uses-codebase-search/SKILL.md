# USES Codebase Search

Use this skill for questions about the local USES/UFT codebase under `/Users/songzuoqiang/Documents/agent/code/uses_codes`.

## Preferred workflow

1. Check whether the local API is already running:
   - `curl -s http://127.0.0.1:8000/health`
2. If it is not running, start it from `/Users/songzuoqiang/Documents/agent/condex/codes`:
   - `PYTHONPATH=src python3 -m uses_indexer serve-api --db /Users/songzuoqiang/Documents/agent/condex/codes/examples/uses_codes_index.db --host 127.0.0.1 --port 8000`
3. For a final grounded answer, call:
   - `POST http://127.0.0.1:8000/answer`
4. If the caller only needs evidence or prompt material, use:
   - `POST http://127.0.0.1:8000/ask`
   - `POST http://127.0.0.1:8000/evidence`

## Response expectations

- Prefer `/answer` when the user is asking a direct repository question.
- Use the returned `final_answer` as the main answer.
- Cite supporting `procedure_name`, `file_path`, and `line_start` / `line_end`.
- If `answer_source` is `draft`, tell the user it is grounded in indexed evidence but not generated by an external model.
- If the question is broad, summarize the top evidence blocks before giving the conclusion.

## Example request

```bash
curl -s http://127.0.0.1:8000/answer \
  -H 'Content-Type: application/json' \
  -d '{"question":"证券代码获取的逻辑在哪里","evidence_limit":3}'
```
